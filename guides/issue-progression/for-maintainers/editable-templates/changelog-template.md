# Changelog
The Issue Progression Initiative recommends the following changelog template, if a changelog will be required for all contributors.

This style is easy to maintain with tens of developers working simultaneously.

## [Unreleased]

### Src
For changes, fixes or additions to source code.

### Tests
For changes, fixes or additions to unit, integration or tck tests.

### Examples
For changes, fixes or additions to documentation.

### .github
For changes, fixes or additions to .github file.

### Packages
For updates to packages.
